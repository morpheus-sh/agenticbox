Run locally in under 2 minutes

# Clone & setup
$ git clone https://github.com/agenticbox/agenticbox.git
$ cd agenticbox && ./scripts/setup.sh

# Start daemon + desktop
$ ./scripts/dev.sh

# Or daemon only (for API access)
$ target/release/daemon
→ Daemon running on http://127.0.0.1:8080

Features by Phase

Sandbox Lifecycle Shipped

Container-based execution with full lifecycle management. Agents run in isolated Docker containers that start and stop on demand.

Terminal Tool Shipped

Execute shell commands inside the sandbox with timeout controls and output capture.

Filesystem Access + Guards Shipped

Read/write files with path resolution guards preventing escapes. Configurable allowed roots.

Permission System Shipped

Per-agent permissions: terminal, filesystem (RO/RW), browser, network (allowlist/localhost/offline).

Session Persistence Shipped

SQLite-backed session management with model config, permissions, and status tracking.

Tauri Desktop App Shipped

Native desktop UI (no Electron) for managing agents, viewing sessions, and monitoring.

Browser Automation (Playwright) In Dev

Headless browser sessions for web interaction. Agents navigate, click, type, and extract data.

Secret Management Planned

API keys, tokens, and credentials — injected securely at runtime without exposure.

Observability (Logs/Metrics/Traces) Planned

Per-agent logs, metrics, and traces. See what agents do, how long it takes, and costs.

Cost Controls Planned

Per-agent billing, usage quotas, and budget alerts for production deployments.

Firecracker MicroVMs Future

Lightweight microVMs for stronger isolation and faster cold starts than containers.

Policy Engine + Multi-Agent Orchestration Future

Advanced policy evaluation and coordinated multi-agent workflows.
