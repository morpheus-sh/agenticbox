The governance layer

[01]

Permissions

What the agent is allowed to do. Terminal, filesystem, network, browser — scoped and enforced. This is what makes it safe to let an agent refund a customer or modify a production record.

[02]

Accountability

Every action attributed, logged, auditable. Full audit trail. This is what makes it compliant to let an agent operate in regulated environments.

[03]

Ownership Boundaries

Clear boundaries: resources, outputs, budgets, assets. What belongs to the agent vs. the organization.

[04]

Identity (emerging)

Agents get their own credentials, accounts, digital identity — provisioned and revocable. The moat that compounds silently as agents accumulate trust.

// What's running today

The runtime

Sandbox Execution

Shipped

Container-based execution with full lifecycle management. Agents run in isolated containers (Docker or Podman, auto-detected) that start and stop on demand.

Execution Permissions

Shipped

Per-agent permissions: terminal, filesystem (RO/RW), browser, network (allowlist/localhost/offline).

Filesystem Governance

Shipped

Read/write files with path resolution guards preventing escapes. Configurable allowed roots per agent.

Session Persistence

Shipped

SQLite-backed session management with model config, permissions, and status tracking across restarts.

Terminal Access

Shipped

Execute shell commands inside the sandbox with timeout controls, output capture, and PTY support.

Native Desktop Console

Shipped

Tauri desktop UI (no Electron) for managing agents, viewing sessions, and monitoring resources.

Agent Packages

Shipped

Run agents like Docker containers: agenticbox run hermes. TOML manifests — shareable, forkable.
